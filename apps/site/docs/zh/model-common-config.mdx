# 常用模型配置

## 配置环境变量的方式

请将所有模型配置项放置在系统环境变量中，Midscene 会自动读取这些环境变量。

以下介绍一些常见方法，你也可以使用自己项目中的其他配置方案。

### 方法一：在系统中设置环境变量

> 在 Midscene Chrome 插件中，你也可以使用这种 `export KEY="value"` 配置格式

```bash
# 替换为你自己的 API Key
export MIDSCENE_MODEL_BASE_URL="https://.../compatible-mode/v1"
export MIDSCENE_MODEL_API_KEY="sk-abcde..."
export MIDSCENE_MODEL_NAME="qwen3-vl-plus"
export MIDSCENE_MODEL_FAMILY="qwen3-vl"
```

### 方法二：编写 `.env` 文件（适用于命令行工具）

在项目的运行路径下创建一个 `.env` 文件，并添加以下内容，Midscene 的命令行工具默认会读取这个文件。

```bash
MIDSCENE_MODEL_BASE_URL="https://.../compatible-mode/v1"
MIDSCENE_MODEL_API_KEY="sk-abcdefghijklmnopqrstuvwxyz"
MIDSCENE_MODEL_NAME="qwen3-vl-plus"
MIDSCENE_MODEL_FAMILY="qwen3-vl"
```

请注意：
1. 这里不需要在每一行前添加 `export`
2. 只有 Midscene 命令行工具会默认读取这个文件，如果是 JavaScript SDK，请参考下一条自行手动加载

### 方法三：引用 dotenv 库配置环境变量

[Dotenv](https://www.npmjs.com/package/dotenv) 是一个零依赖的 npm 包，用于将环境变量从 `.env` 文件加载到 node.js 的环境变量参数 `process.env` 中。

我们的 [demo 项目](https://github.com/web-infra-dev/midscene-example) 使用了这种方式。

```bash
# 安装 dotenv
npm install dotenv --save
```

在项目根目录下创建一个 `.env` 文件，并添加以下内容。注意这里不需要在每一行前添加 `export`。

```bash
MIDSCENE_MODEL_API_KEY="sk-abcdefghijklmnopqrstuvwxyz"
```

在脚本中导入 dotenv 模块，导入后它会自动读取 `.env` 文件中的环境变量。

```typescript
import 'dotenv/config';
```


## 常用模型配置

这里列出常用模型的配置，如需了解模型区别和选型，可查阅 [推荐的视觉模型](./model-strategy#recommended-vision-models)。

### 豆包 Seed 模型 {#doubao-seed-model}

推荐使用 Doubao-Seed-2.0-Lite，同时也支持 Doubao-Seed-1.6-Vision 和 Doubao-Seed-1.8 模型。

从 [火山引擎](https://volcengine.com) 获取 API 密钥，然后补充以下环境变量：

```bash
MIDSCENE_MODEL_BASE_URL="https://ark.cn-beijing.volces.com/api/v3"
MIDSCENE_MODEL_API_KEY="...."
MIDSCENE_MODEL_NAME="ep-..." # 来自火山引擎的推理接入点 ID 或模型名称
MIDSCENE_MODEL_FAMILY="doubao-seed" # 也兼容 "doubao-vision"
```

如果你的火山引擎账号已开通低延迟模式（Fast），可以追加以下请求体参数来使用该能力。通常可将模型响应速度提升约 30%-50%。

```bash
MIDSCENE_MODEL_EXTRA_BODY_JSON={"service_tier":"fast"}
```

Midscene 会默认关闭模型原生思考以获得最佳的执行速度和执行效果，见[模型原生的思考模式](./model-strategy#模型原生的思考模式)。如果需要开启原生的思考模式，可以配置 `MIDSCENE_MODEL_REASONING_ENABLED="true"`，并通过 `MIDSCENE_MODEL_REASONING_EFFORT="medium"` 控制推理力度。

### 千问 Qwen3.x 系列 {#qwen3x}

目前已支持 Qwen3.x 系列中的 Qwen3.5、Qwen3.6、Qwen3.7，推荐程度：Qwen3.7 > Qwen3.5> Qwen3.6。

以[阿里云](https://www.aliyun.com/)的 `qwen3.7-plus` 模型为例，其环境变量配置如下：

```bash
MIDSCENE_MODEL_BASE_URL="https://dashscope.aliyuncs.com/compatible-mode/v1"
MIDSCENE_MODEL_API_KEY="......"
MIDSCENE_MODEL_NAME="qwen3.7-plus" # Qwen3.5 和 Qwen3.6 系列的 plus 模型分别为 "qwen3.5-plus"、"qwen3.6-plus"
MIDSCENE_MODEL_FAMILY="qwen3" // 对于 Qwen3.5 和 Qwen3.6，family 曾使用 `qwen3.5`、`qwen3.6`，依然兼容。
```

Midscene 会默认关闭模型原生思考以获得最佳的执行速度和执行效果，见[模型原生的思考模式](./model-strategy#模型原生的思考模式)。如果需要开启原生的思考模式，可以配置 `MIDSCENE_MODEL_REASONING_ENABLED="true"`，并通过 `MIDSCENE_MODEL_REASONING_BUDGET="500"` 控制思考成本。

### 智谱 GLM-V 系列 {#glm-v}

智谱 GLM-V 是智谱 AI 推出的视觉理解模型。最新版本有 `GLM-4.6V`（开源）、`GLM-5V-Turbo`。

以 `GLM-4.6V` 为例：

从 [Z.AI（国际）](https://z.ai/manage-apikey/apikey-list)或 [BigModel（国内）](https://bigmodel.cn/usercenter/proj-mgmt/apikeys)获取 API 密钥，然后设置：

```bash
MIDSCENE_MODEL_BASE_URL="https://open.bigmodel.cn/api/paas/v4" # 或 https://api.z.ai/api/paas/v4
MIDSCENE_MODEL_API_KEY="......"
MIDSCENE_MODEL_NAME="glm-4.6v" # GLM-5V-Turbo 的对应模型名为 "glm-5v-turbo"
MIDSCENE_MODEL_FAMILY="glm-v"
```

Midscene 会默认关闭模型原生思考以获得最佳的执行速度和执行效果，见[模型原生的思考模式](./model-strategy#模型原生的思考模式)。如果需要开启原生的思考模式，可以配置 `MIDSCENE_MODEL_REASONING_ENABLED="true"`。

**了解更多关于智谱 GLM-V 开源模型**

- Github: [https://github.com/zai-org/GLM-V](https://github.com/zai-org/GLM-V)
- Hugging Face: [https://huggingface.co/zai-org/GLM-4.6V](https://huggingface.co/zai-org/GLM-4.6V)


### 智谱 AutoGLM {#auto-glm}

智谱 AutoGLM 是智谱 AI 推出的开源移动端 UI 自动化模型，模型尺寸为 9B。

从 [Z.AI（国际）](https://z.ai/manage-apikey/apikey-list)或 [BigModel（国内）](https://bigmodel.cn/usercenter/proj-mgmt/apikeys)获取 API 密钥后，可以使用以下配置：

```bash
MIDSCENE_MODEL_BASE_URL="https://open.bigmodel.cn/api/paas/v4" # 或 https://api.z.ai/api/paas/v4
MIDSCENE_MODEL_API_KEY="......"
MIDSCENE_MODEL_NAME="autoglm-phone" # 模型名以平台实际模型名为准
MIDSCENE_MODEL_FAMILY="auto-glm" # 或 "auto-glm-multilingual"
```

**关于 `MIDSCENE_MODEL_FAMILY` 配置**

AutoGLM 提供了两个版本的模型，通过 `MIDSCENE_MODEL_FAMILY` 区分：

- `auto-glm` - 对应 AutoGLM-Phone-9B，针对**中文环境**优化
- `auto-glm-multilingual` - 对应 AutoGLM-Phone-9B-Multilingual，支持**英语等其他语言**场景

请根据你的应用语言选择合适的版本。

:::note
AutoGLM 更适合移动端的交互与操作流程。如果要使用 `aiAssert`、`aiQuery` 等需要页面理解/断言的 API，请额外配置一组 `MIDSCENE_INSIGHT_MODEL_...` 环境变量，让独立的 Insight 模型负责页面理解，具体可参考[模型策略](./model-strategy) 中对于多模型配合的介绍。
:::

**了解更多关于智谱 AutoGLM**

- Github: [https://github.com/zai-org/Open-AutoGLM](https://github.com/zai-org/Open-AutoGLM)
- Hugging Face: [https://huggingface.co/zai-org/AutoGLM-Phone-9B](https://huggingface.co/zai-org/AutoGLM-Phone-9B)

### Gemini 3.x 系列 {#gemini-3-pro}

Gemini 3.x 系列中的 `gemini-3.5-flash` 是目前我们私有测评集中定位表现最好的模型。在 [Google Gemini](https://gemini.google.com/) 上申请 API Key 后，可以使用以下配置。

`MIDSCENE_MODEL_NAME` 请填写你使用的 Gemini 3.x 系列具体模型名：

```bash
MIDSCENE_MODEL_BASE_URL="https://generativelanguage.googleapis.com/v1beta/openai/"
MIDSCENE_MODEL_API_KEY="......"
MIDSCENE_MODEL_NAME="gemini-3.5-flash" # 或具体的 Gemini 3.x 模型名
MIDSCENE_MODEL_FAMILY="gemini"
```

### GPT-5 系列 {#gpt-5-4}

在 OpenAI 或兼容 OpenAI 接口的平台上使用 GPT-5 系列模型时，可以参考如下配置：

```bash
MIDSCENE_MODEL_BASE_URL="https://api.openai.com/v1" # 或你的兼容服务地址
MIDSCENE_MODEL_API_KEY="sk-..."
MIDSCENE_MODEL_NAME="gpt-5.5" # 推荐 gpt-5.5 或 gpt-5.4
MIDSCENE_MODEL_FAMILY="gpt-5"
```

:::warning GPT-5 使用注意事项

- 使用 GPT 做 UI 定位时，目前只支持使用 `gpt-5.4`、`gpt-5.5`。因为为了获得最佳的定位效果，需要在发送图片时指定 `"detail": "original"` 参数，这一参数仅在 `gpt-5.4` 及后续模型上可用，`gpt-5.4-mini`、`gpt-5.4-nano` 等更小的 GPT-5 变体以及前代模型不支持 `original` 参数，会导致报错。详情请参考 [Images and Vision guide](https://developers.openai.com/api/docs/guides/images-vision) 和 [Computer use guide](https://developers.openai.com/api/docs/guides/tools-computer-use)。
- 按照 OpenAI 的文档，GPT-5 在处理非拉丁字母文本、字号太小的文本时效果可能不理想，参见 [Images and Vision guide](https://developers.openai.com/api/docs/guides/images-vision)。
- OpenAI 在 computer use 文档中提到，他们观察到 `1440x900` 和 `1600x900` 这两种截图尺寸上通常能获得比较好的效果，详见 [Computer use guide](https://developers.openai.com/api/docs/guides/tools-computer-use)。因此，建议按照 OpenAI 的推荐对截图尺寸进行调整。在 Midscene 中，你可以通过 Agent 参数里的 `screenshotShrinkFactor` 控制截图压缩倍率。如果是浏览器自动化，还可以通过浏览器 `viewport` 指定页面的尺寸和比例。
- 如果你使用的是更老版本的 GPT-5，建议只将其用作规划模型，并搭配其他视觉模型完成定位，参考[多模型示例](#gpt54-planning-insight-qwen35)。

:::

## 自定义模型适配器（实验性） {#custom-model-adapter}

:::warning 实验性功能

自定义模型适配器是一个实验性功能，可以通过 `MIDSCENE_MODEL_FAMILY` 指定一段自定义的 `.cjs` adapter 文件，控制 Midscene 对模型的兼容逻辑。

通常情况下，你应该优先使用 Midscene 内置的 `MIDSCENE_MODEL_FAMILY`。该功能仅用于临时快速验证某款模型是否适用于 Midscene，且随着 Midscene 的版本迭代，该功能的接口和配置方式随时可能调整。

:::

### 基本配置

将 `MIDSCENE_MODEL_FAMILY` 修改为 `custom:<specifier>` 启用自定义模型适配器：

```bash
MIDSCENE_MODEL_BASE_URL="https://your-endpoint.example.com/v1"
MIDSCENE_MODEL_API_KEY="your-api-key"
MIDSCENE_MODEL_NAME="your-model-name"
MIDSCENE_MODEL_FAMILY="custom:./midscene-adapter.cjs"
```

这里的 `custom:` 是一个自定义 Model Adapter 引用，`custom:` 后面需要指定一个相对路径或绝对路径的 CommonJS 文件。

### 编写 Adapter 文件

你可以参考如下的示例定义 adapter 文件，对于大部分的模型，只需要适配下面两部分字段：

- `chatCompletion.buildChatCompletionParams`：把 Midscene 的模型配置转换成当前模型服务需要的 chat completion 参数。
- `locate.resultAdapter.coordinates`：声明模型返回的坐标格式。

```js
// midscene-adapter.cjs
module.exports = {
  chatCompletion: {
    buildChatCompletionParams: ({ userConfig, midsceneDefaults }) => {
      const commonOverrideConfig = {};

      if (userConfig.temperature !== undefined) {
        commonOverrideConfig.temperature = userConfig.temperature;
      }

      // 适配模型的思考参数。如果用户没有自定义思考配置，Midscene 推荐默认关闭模型思考。
      // 字段名请以模型服务文档为准。这里以 Qwen 兼容接口为例。
      const reasoningEnabled = userConfig.reasoningEnabled ?? false;
      const modelSpecificConfig = {
        enable_thinking: reasoningEnabled,
      };

      // 可以在这里定义任何传给模型的 chat 参数，比如思考力度、思考额度。
      if (userConfig.reasoningBudget !== undefined) {
        modelSpecificConfig.thinking_budget = userConfig.reasoningBudget;
      }

      return {
        config: {
          ...midsceneDefaults,
          ...commonOverrideConfig,
          ...modelSpecificConfig,
        },
      };
    },
  },

  locate: {
    resultAdapter: {
      coordinates: {
        shape: 'bbox',
        order: 'xy',
        normalizedBy: 1000,
      },
    },
  },
};
```

`buildChatCompletionParams` 的入参中可以拿到当前调用的上下文，有以下参数：

- `userConfig`：用户通过环境变量定义的参数，目前有：
  - `temperature`：对应 `MIDSCENE_MODEL_TEMPERATURE`，用于控制模型采样温度。
  - `reasoningEnabled`：对应 `MIDSCENE_MODEL_REASONING_ENABLED`，用于控制是否开启模型原生思考。
  - `reasoningBudget`：对应 `MIDSCENE_MODEL_REASONING_BUDGET`，用于控制模型原生思考的 token 预算。
- `midsceneDefaults`：Midscene 默认的 chat completion 参数，可使用自定义的配置覆盖。

建议默认关闭模型原生思考，以获得最佳的执行速度和执行效果。也就是说，当 `userConfig.reasoningEnabled` 没有显式配置时，可以按 `false` 处理；如果用户显式设置为 `true`，再把对应的思考参数传给模型服务。不同模型服务关闭思考的字段名不同，例如有的使用 `enable_thinking: false`，有的使用 `thinking: { type: 'disabled' }`，请以实际模型服务文档为准。

`coordinates` 用来描述模型返回的坐标格式：

- `shape`：模型返回的坐标形状。`'bbox'` 表示模型返回矩形框，`'point'` 表示模型返回点坐标。默认为 `'bbox'`。
- `order`：模型返回的坐标顺序。`'xy'` 表示坐标顺序为 `[x, y]`，对于 bbox 则为 `[xmin, ymin, xmax, ymax]`；`'yx'` 表示坐标顺序为 `[y, x]`，对于 bbox 则为 `[ymin, xmin, ymax, xmax]`。
- `normalizedBy`：坐标的归一化基准。如果模型返回的是 0 到 1000 的归一化坐标，则配置 `normalizedBy: 1000`；`undefined` 则表示模型返回像素坐标。

### 路径和导出规则

当前自定义模型适配器有以下限制：

- `custom:` 后面需要指定一个相对路径或绝对路径的 CommonJS 文件。
- 仅支持 `.cjs` 文件，并且必须通过 `module.exports` 导出 adapter 对象。
- 不支持 `.mjs`、ESM module、TypeScript 源文件或 async factory。
- 该能力主要面向 Node.js runtime，不保证在浏览器、Chrome extension bundle 等环境中可用。

### UI-TARS {#ui-tars}

你可以在 [火山引擎](https://volcengine.com) 上使用已部署的 `doubao-1.5-ui-tars`。

```bash
MIDSCENE_MODEL_BASE_URL="https://ark.cn-beijing.volces.com/api/v3"
MIDSCENE_MODEL_API_KEY="...."
MIDSCENE_MODEL_NAME="ep-2025..." # 来自火山引擎的推理接入点 ID 或模型名称
MIDSCENE_MODEL_FAMILY="vlm-ui-tars-doubao-1.5"
```

**关于 `MIDSCENE_MODEL_FAMILY` 配置**

`MIDSCENE_MODEL_FAMILY` 用于指定 UI-TARS 版本，使用以下值之一：
- `vlm-ui-tars` - 用于模型版本 `1.0`
- `vlm-ui-tars-doubao` - 用于在火山引擎上部署的模型版本 `1.5`（与 `vlm-ui-tars-doubao-1.5` 等效）
- `vlm-ui-tars-doubao-1.5` - 用于在火山引擎上部署的模型版本 `1.5`

:::tip

旧版本使用 `MIDSCENE_USE_VLM_UI_TARS=DOUBAO` 或 `MIDSCENE_USE_VLM_UI_TARS=1.5` 配置，该配置仍然兼容但已废弃，建议迁移到 `MIDSCENE_MODEL_FAMILY`。

迁移对应关系：
- `MIDSCENE_USE_VLM_UI_TARS=1.0` → `MIDSCENE_MODEL_FAMILY="vlm-ui-tars"`
- `MIDSCENE_USE_VLM_UI_TARS=1.5` → `MIDSCENE_MODEL_FAMILY="vlm-ui-tars-doubao-1.5"`
- `MIDSCENE_USE_VLM_UI_TARS=DOUBAO` → `MIDSCENE_MODEL_FAMILY="vlm-ui-tars-doubao"`

:::

## 多模型示例：GPT-5.4 规划/理解 + Qwen 3.5 视觉定位 {#gpt54-planning-insight-qwen35}

关于组合多个模型的更多信息，可查阅 [进阶：组合多个模型](./model-strategy#advanced-combining-multiple-models)。

下面以 GPT-5.4 用于 Planning/Insight、Qwen 3.5 负责视觉为例。使用 GPT-5.4 处理重度推理（Planning 和/或 Insight），让 Qwen 3.5 专注视觉定位。独立的 Planning 和 Insight 模型可按需启用，不需要同时开启。

```bash
# 默认视觉模型：Qwen 3.5
export MIDSCENE_MODEL_BASE_URL="https://..."       # Qwen 3.5 接口地址
export MIDSCENE_MODEL_API_KEY="..."                # 你的 Qwen 3.5 API Key
export MIDSCENE_MODEL_NAME="qwen3.5-plus"
export MIDSCENE_MODEL_FAMILY="qwen3.5"

# Planning 模型：GPT-5.4
export MIDSCENE_PLANNING_MODEL_API_KEY="sk-..."    # 你的 GPT-5.4 API Key
export MIDSCENE_PLANNING_MODEL_BASE_URL="https://..."
export MIDSCENE_PLANNING_MODEL_NAME="gpt-5.4"
export MIDSCENE_PLANNING_MODEL_FAMILY="gpt-5"

# Insight 模型：GPT-5.4
export MIDSCENE_INSIGHT_MODEL_API_KEY="sk-..."     # 你的 GPT-5.4 API Key
export MIDSCENE_INSIGHT_MODEL_BASE_URL="https://..."
export MIDSCENE_INSIGHT_MODEL_NAME="gpt-5.4"
export MIDSCENE_INSIGHT_MODEL_FAMILY="gpt-5"
```
