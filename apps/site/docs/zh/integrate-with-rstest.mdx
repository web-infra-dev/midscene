# 集成到 Rstest

[Rstest](https://github.com/web-infra-dev/rstest) 是基于 Rspack 的高性能测试框架。通过 `@midscene/rstest`，你可以用自然语言描述浏览器操作和断言，并把它们当作普通的 Rstest 测试来运行：

```ts
import { expect, test } from '@midscene/rstest/playwright';

test('adds a todo', async ({ agent, page }) => {
  await page.goto('http://localhost:5173/');
  await agent.aiAct("type 'Study AI' in the input and press Enter");
  await agent.aiAssert('the list contains exactly one item: "Study AI"');
});
```

Midscene 会启动浏览器、为每个测试创建 agent，并为每个测试文件生成一份可视化 HTML 报告，你可以在报告中带着截图回放每一步 AI 操作。

`@midscene/rstest` 构建在 [`@rstest/playwright`](https://github.com/web-infra-dev/rstest/tree/main/packages/playwright) 之上，它提供的能力在这里全部可用：Playwright 风味的 `expect`（`toBeVisible`、`toHaveText` 等）、debug 模式（`PWDEBUG`）、trace 采集，以及 `request` / `serve` fixtures。

:::info 仅支持 Playwright

`@midscene/rstest` 通过 Playwright 驱动浏览器，所以 import 路径是 `@midscene/rstest/playwright`。Puppeteer 不支持。

:::

## 第一步：安装依赖

安装 `@midscene/rstest`，以及 `@midscene/web`、`@rstest/core`、`@rstest/playwright` 和 `playwright`：

<PackageManagerTabs command="install @midscene/web @midscene/rstest @rstest/core @rstest/playwright playwright --save-dev" />

:::tip 版本要求

- `@rstest/core >= 0.11.2`
- `playwright >= 1.49.1`
- Node.js `^20.19.0 || >=22.12.0`

:::

## 第二步：配置 Rstest

在测试项目根目录创建 `rstest.config.ts`：

```ts title="rstest.config.ts"
import 'dotenv/config'; // load model env vars from .env
import MidsceneReporter from '@midscene/rstest/reporter';
import { defineConfig } from '@rstest/core';

export default defineConfig({
  include: ['e2e/**/*.test.ts'],
  // AI calls take time, so leave plenty of headroom for tests and hooks.
  testTimeout: 1_800_000,
  hookTimeout: 60_000,
  reporters: ['default', new MidsceneReporter()],
});
```

在 `reporters` 中注册 `MidsceneReporter`。每个测试文件跑完后，它会把该文件中所有测试的报告合并成一份 HTML 报告，并在控制台打印路径。

## 第三步：编写测试

从 `@midscene/rstest/playwright` 导入 `test` 和 `expect`，在测试函数中解构需要的 fixture：

```ts title="e2e/todo-list.test.ts"
import { expect, test } from '@midscene/rstest/playwright';

test('adds and completes a todo', async ({ agent, page }) => {
  await page.goto('http://localhost:5173/');
  await agent.aiAct("type 'Study AI' in the input and press Enter");
  await agent.aiAct('click the checkbox of the first item');

  const list = await agent.aiQuery<string[]>(
    'string[], the complete task list',
  );
  expect(list).toHaveLength(1);

  await agent.aiAssert('the only item is marked as done');
});
```

在测试开头调用 `page.goto(...)` 打开要测试的页面，然后用 `agent` 与页面交互。所有 agent 方法见 [API 参考](./api#交互方法)。

可用的 fixtures：

| Fixture | 说明 |
| --- | --- |
| `agent` | 当前页面的 Midscene agent。用 `agent.aiAct(...)`、`agent.aiQuery(...)`、`agent.aiAssert(...)` 等方法驱动和检查页面。 |
| `page` | Playwright `Page`。用于导航（`page.goto`），也可以调用任意 Playwright API，比如 `page.route(...)`、`page.evaluate(...)`。 |
| `browser` | 共享的 Playwright `Browser`。首次使用时启动，自动关闭。用于开额外的 context 或 page。 |
| `context` | `page` 所属的 `BrowserContext`。 |
| `request` / `serve` | 来自 `@rstest/playwright`：隔离的 `APIRequestContext` 用于发 HTTP 请求，以及一行代码起静态文件服务器。 |
| `agentForPage(page, opts?)` | 为另一个页面（比如 popup 或第二个会话）创建 Midscene agent，操作会记录进同一份报告。 |
| `playwright` | 浏览器选项：launch、context、debug、trace。见 [配置浏览器](#配置浏览器)。 |
| `agentOptions` | Agent 选项：cache、action context 等。见 [配置 agent](#配置-agent)。 |

AI 方法和 Playwright API 可以自由混用。比如用 `page.route` mock 接口，再用 `aiAssert` 验证结果：

```ts
test('mocks the API and verifies the UI', async ({ agent, page }) => {
  await page.route('**/api/contacts', (route) =>
    route.fulfill({ json: [{ name: 'Fake Alice' }] }),
  );
  await page.goto('http://localhost:5173/');
  await agent.aiAssert('the contact list shows "Fake Alice"');
});
```

## 第四步：运行测试

```bash
npx rstest run

# 或在开发时：
npx rstest watch
```

每个测试文件跑完后，控制台会打印报告路径：

```text
Midscene report: midscene_run/report/<file>.html
```

在浏览器中打开即可带着截图回放每一步 AI 操作。如需以编程方式处理报告文件，见 [处理报告文件](./consume-report-file)。

## 配置浏览器

浏览器相关行为——launch options、context options、debug、trace——通过来自 `@rstest/playwright` 的 `playwright` fixture 配置，[上游文档](https://github.com/web-infra-dev/rstest/tree/main/packages/playwright)中的每个选项都可用：

```ts
import { test as base } from '@midscene/rstest/playwright';

const test = base.extend({
  playwright: {
    launchOptions: { headless: false },
    contextOptions: { locale: 'zh-CN', storageState: 'auth.json' },
    debug: true, // headed + slowMo + DevTools, pause on failure (also: PWDEBUG=1)
    trace: 'retain-on-failure', // save trace.zip (also: RSTEST_PLAYWRIGHT_TRACE)
  },
});
```

`@midscene/rstest` 为这个 fixture 提供了默认值——CI 中 headless、本地 headed，viewport 为 1920×1080。覆盖 fixture 时整个对象会被替换，想保留默认值就 spread `defaultPlaywrightOptions`：

```ts
import {
  test as base,
  defaultPlaywrightOptions,
} from '@midscene/rstest/playwright';

const test = base.extend({
  playwright: {
    ...defaultPlaywrightOptions,
    contextOptions: { viewport: { width: 375, height: 667 } },
  },
});
```

保存的 trace 位于 `.rstest/playwright-traces/`，控制台会打印 `npx playwright show-trace ...` 命令用于打开。

## 配置 agent

`agentOptions` fixture 设置测试中构造每个 agent 时使用的选项，接受与 `PlaywrightAgent` 相同的选项，比如 `aiActionContext`、`modelConfig`、`waitAfterAction`、`replanningCycleLimit`：

```ts
const test = base.extend({
  agentOptions: {
    aiActionContext: 'Close any promo dialog first.',
    cache: true,
  },
});
```

设置 `cache: true` 可以在多次运行间复用 AI 规划结果，加速重复执行。Midscene 会从测试文件名和测试名派生一个稳定的 cache id，重跑同一个测试会命中同一份缓存。如需自己控制策略或 id，传 `cache: { strategy, id }`。详见 [缓存](./caching)。

`agentForPage(page, opts)` 每次调用也接受同样的选项，优先级高于 `agentOptions`。

## 设置项目级默认值

想让所有测试文件使用同一套选项，在共享模块中扩展基础 test，并在各处导入扩展后的 `test`：

```ts title="tests/midscene.ts"
import {
  test as base,
  defaultPlaywrightOptions,
} from '@midscene/rstest/playwright';

export { describe, expect } from '@midscene/rstest/playwright';

export const test = base.extend({
  playwright: {
    ...defaultPlaywrightOptions,
    contextOptions: { locale: 'en-US', viewport: { width: 1280, height: 720 } },
  },
  agentOptions: { cache: true },
});
```

```ts title="tests/todo.test.ts"
import { expect, test } from './midscene';
```

需要不同配置的文件再次调用 `extend` 即可——后面的值替换前面的值。

## 自定义 page

想让一个文件里的所有测试打开同一个 URL，或需要在测试开始前对页面做准备——init script、storage state 等——可以覆盖 `page` fixture。`await use(page)` 之前的代码在每个测试前运行，之后的代码作为 teardown 运行：

```ts
import type { Page } from 'playwright';
import { test as base } from '@midscene/rstest/playwright';

const test = base.extend<{ page: Page }>({
  page: async ({ context }, use) => {
    await context.addInitScript(() => {
      window.localStorage.setItem('feature-flag', 'on');
    });
    const page = await context.newPage();
    await page.goto('https://example.com');

    await use(page);

    await page.close();
  },
});
```

你的 teardown 代码运行时，Midscene 已经收集完报告，页面也仍然打开着——适合在这里保存录屏或导出 cookies。

需要更深层的定制（比如 persistent context）时，改为覆盖 `context`；默认的 `page` 会基于你提供的 context 创建。注意 fixture 覆盖读不到被替换的原值：自定义 `page` 要从 `context` 构建，自定义 `context` 要从 `browser` 构建。

## 测试多个页面或多个用户

用 `agentForPage` 为主页面之外的任何页面获取 agent，操作会记录进同一份报告，清理自动完成。

驱动 popup：

```ts
test('drives a popup with a second agent', async ({
  agent,
  page,
  agentForPage,
}) => {
  await page.goto('http://localhost:5173/');
  await agent.aiTap("the 'View details' button");
  const popup = await page.waitForEvent('popup');

  const popupAgent = await agentForPage(popup);
  await popupAgent.aiAssert('the popup shows order details');
});
```

在一个测试里跑两个已登录用户——从 `browser` 开第二个 context（有独立的 cookies 和 storage）：

```ts
test('user B receives the message from user A', async ({
  agent,
  page,
  browser,
  agentForPage,
}) => {
  await page.goto('http://localhost:5173/');

  const sessionB = await browser.newContext({ storageState: 'user-b.json' });
  const pageB = await sessionB.newPage();
  await pageB.goto('http://localhost:5173/');
  const agentB = await agentForPage(pageB);

  await agent.aiAct("send 'hello' to user B");
  await agentB.aiWaitFor("the chat shows a new message 'hello'");
});
```

## 用自定义 fixture 复用 setup

在 Rstest 中，`beforeEach` 等 hook 无法访问 fixture。共享的 setup——登录、准备数据、准备第二个角色——通过 `test.extend` 写成自定义 fixture，你的 fixture 可以依赖 `agent` 和 `agentForPage`。

让文件里的每个测试先登录——用 `{ auto: true }` 声明 fixture，即使测试没有引用它也会执行：

```ts title="e2e/fixtures.ts"
import { test as base } from '@midscene/rstest/playwright';

export const test = base.extend<{ loggedIn: void }>({
  loggedIn: [
    async ({ page, agent }, use) => {
      await page.goto('http://localhost:5173/');
      await agent.aiAct('log in with alice / password');
      await use(undefined);
    },
    { auto: true },
  ],
});

export { expect } from '@midscene/rstest/playwright';
```

添加一个随时可用的第二角色，带自己的登录态：

```ts
import type { PlaywrightAgent } from '@midscene/web/playwright';

const test = base.extend<{ adminAgent: PlaywrightAgent }>({
  adminAgent: async ({ browser, agentForPage }, use) => {
    const context = await browser.newContext({ storageState: 'admin.json' });
    const page = await context.newPage();
    await page.goto('http://localhost:5173/');
    await use(await agentForPage(page));
    await context.close();
  },
});

test('only admin can delete', async ({ agent, page, adminAgent }) => {
  await page.goto('http://localhost:5173/');
  await agent.aiAssert('no Delete button is visible');
  await adminAgent.aiAssert('a Delete button is visible');
});
```

`agent` 和 `agentForPage` 本身无法在 `test.extend` 中覆盖——TypeScript 会报错，因为替换后会丢失报告收集。像上面这样在自定义 fixture 中依赖它们即可。

## 配置模型

模型接入通过环境变量配置——API 地址、密钥、模型名等。写在 `.env`、CI secrets 或 shell 导出中即可自动生效。完整列表见[模型配置](./model-config)。

```bash
MIDSCENE_MODEL_BASE_URL=https://api.openai.com/v1
MIDSCENE_MODEL_API_KEY=sk-...
MIDSCENE_MODEL_NAME=qwen3.7-plus
```

想在运行时修改模型配置——比如根据环境切换模型——在 setup 文件中调用 `overrideAIConfig`：

```ts title="midscene.setup.ts"
import { overrideAIConfig } from '@midscene/rstest/playwright';

overrideAIConfig({
  MIDSCENE_MODEL_NAME: process.env.CI ? 'gpt-5.5' : 'gpt-5.4',
});
```

打印 debug 日志用 `DEBUG` 环境变量：

```bash
DEBUG=midscene:*,rstest:* npx rstest run
```
