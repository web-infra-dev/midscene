# BDD-style scripts with Gherkin

> This feature is experimental. The API may change in future releases.
>
> Supported since Midscene 1.10.

In AI-driven GUI automation, Gherkin is a natural fit because each GUI operation can be written as a behavior-oriented natural language step while still preserving a clear test structure.

Gherkin is a plain-text syntax for describing behavior with examples. It is often used in BDD workflows, where a scenario is written as a sequence of steps. See the official [Cucumber Gherkin reference](https://cucumber.io/docs/gherkin/reference) for the full syntax.

```gherkin
Scenario: Add a todo
  Given the todo page is open
  When I add "Buy milk"
  Then the todo list contains "Buy milk"
```

Midscene supports running one Gherkin Scenario as a compact way to write an AI automation flow. The scenario text is parsed into Midscene Agent calls, so you can keep the script readable while still using `aiAct` and `aiAssert` behind the scenes.

## Supported rules

Midscene supports a focused subset of Gherkin:

- `Given` maps to `aiAct`. Midscene treats it as a precondition setup step.
- `When` maps to `aiAct`. Midscene treats it as a user action step.
- `Then` maps to `aiAssert`. Midscene treats it as a verification step.
- `And` and `But` reuse the previous primary keyword. For example, an `And` after `Then` also maps to `aiAssert`. Keywords are matched case-insensitively, so `AND` is also supported.
- Exactly one `Scenario:` is supported. You may also write only steps without a `Scenario:` header.

These Gherkin features are not supported:

- `Feature`
- `Background`
- `Scenario Outline` or `Scenario Template`
- `Examples`
- `Rule`
- data tables
- doc strings
- multiple `Scenario:` blocks in the same input

## Use in YAML

Use the `runGherkinScenario` step in a YAML flow. The value should be a block string containing one scenario.

```yaml
web:
  url: http://localhost:3000/todos

tasks:
  - name: Add a todo item
    flow:
      - runGherkinScenario: |
          Scenario: Add a todo
            Given the todo page is open
            When I add "Buy milk" as a todo item
            Then the todo list contains "Buy milk"
            And the todo count is 1
```

During YAML execution, Midscene runs the scenario step by step. `Given` and `When` steps are executed with `aiAct`, while `Then` and the following `And` step are executed with `aiAssert`.

YAML `runGherkinScenario` intentionally disables `aiAct` cache for the scenario so every Gherkin step is interpreted from the current page state.

You can also write a longer scenario with several Given / When / Then groups. Midscene still runs it as one scenario and executes the steps in order.

```yaml
web:
  url: http://localhost:3000/checkout

tasks:
  - name: Complete checkout
    flow:
      - runGherkinScenario: |
          Scenario: Place an order with a saved address
            Given the shopping cart contains "Midscene Mug"
            When I open the cart page
            Then the cart shows "Midscene Mug"
            And the subtotal is visible

            Given the checkout page is open
            When I choose the saved shipping address
            Then the shipping address section shows the selected address

            Given the payment section is visible
            When I choose the saved credit card and submit the order
            Then the order confirmation page is displayed
            And the confirmation page shows an order number
```

This style is useful when one business scenario naturally contains several UI phases, such as cart review, shipping selection, payment, and final confirmation.
